01

Literature Survey

Cricket analytics has evolved from manual scorecards to sensor-driven performance systems. Recent advances in computer vision — particularly human pose estimation frameworks such as MediaPipe and OpenPose — have enabled non-invasive motion analysis from standard video footage.

Research in sports action recognition leverages convolutional and recurrent neural networks to classify movements from sequential frame data. Transfer learning with architectures like ResNet, EfficientNet, and Vision Transformers has shown strong performance in fine-grained activity recognition tasks relevant to cricket shot identification.

Bowling action legality detection has been studied using marker-based motion capture in laboratory settings. However, markerless video-based approaches remain under-explored, presenting opportunities for scalable, field-deployable solutions.

02

Research Gap

Existing cricket analysis tools are either prohibitively expensive (requiring specialised motion-capture labs) or limited to basic statistics like run rates and economy. There is no integrated, mobile-first platform that combines real-time batting shot classification, biomechanical similarity scoring, bowling performance metrics, and action legality detection in a single pipeline.

Moreover, current similarity-scoring methods rely on hand-crafted features rather than learned representations, limiting their generalisation across diverse playing styles and conditions.

03

Research Problem

How can we build an end-to-end, video-based cricket analysis system that performs shot classification, shot similarity scoring, bowler performance evaluation, and bowling action legality detection — using only standard smartphone camera footage, without specialised hardware or controlled environments?

04

Objectives

1. Develop a shot classification model that accurately categorises batting strokes (drive, cut, pull, sweep, defence, etc.) from video sequences using pose estimation and deep learning.

2. Build a similarity scoring engine that compares a player's shot mechanics against reference professional players, providing quantitative feedback on technique closeness.

3. Create a bowling performance analysis module that extracts delivery speed, length, line, trajectory, and session-level aggregate statistics from video.

4. Implement a bowling action legality detector that estimates elbow extension angles from video to flag potentially illegal bowling actions.

05

Methodology

Data Collection: Match and practice session footage captured using smartphone cameras at standard frame rates. Stump-to-stump calibration enables spatial measurements from 2D video.

Pose Estimation: MediaPipe Pose is used to extract 33 body-landmark coordinates per frame, forming the input feature set for downstream models.

Shot Classification: A temporal convolutional network (TCN) processes sequences of pose landmarks to classify batting shots into predefined categories.

Similarity Scoring: Dynamic Time Warping (DTW) and learned embeddings compare landmark trajectories between a player's shot and reference shots from professional players.

Bowling Analysis: Ball tracking via object detection (YOLOv8) combined with homography-based pitch mapping yields speed, length, and trajectory metrics.

Legality Detection: Elbow joint angles are computed from pose landmarks across the bowling action to detect hyperextension beyond the 15° ICC threshold.

06

Technologies

Frontend: React Native with Expo for the cross-platform mobile application.

Backend: Python FastAPI microservices for video processing, model inference, and data management.

ML / CV: MediaPipe, YOLOv8, PyTorch, OpenCV, scikit-learn.

Infrastructure: Firebase for authentication and storage; deployed on cloud GPU instances for inference.
